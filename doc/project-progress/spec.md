# Project Specification — Cognitive Bias Tester

> Working name. Repository: `cognitive-bias-tester`. Brand and domain TBD before public launch (E130).

## Goal

A web application — *"Duolingo for logical thinking"* — that teaches users to recognize their own cognitive biases through realistic, AI-generated quiz Cases. The system serves three modes: a single-question web flow (Mode 1), a comprehensive parametrizable test for organisations (Mode 2, post-MVP), and an automatic social-media publishing pipeline that drives organic growth (Mode 3, MVP launch).

## Scope (Monetized MVP — Epics E010–E130)

The MVP delivers a publicly launchable, monetised product driven by viral mechanics on a single social network (X / Twitter), with a working free → paid conversion funnel, in two human-curated UI languages (cs + en) and on-demand LLM translation for any other language at the Case / in-app micro-copy level.

**In scope:**

- **Mode 1** (single-question web flow): user sees a Case in their language, answers, gets explanation + bias type, can rate the Case quality (stars), sees aggregated stats (own progress, all users, language-segmented).
- **Mode 3** (social publishing): scheduled posts on X with automatically generated Open Graph preview images; deep-link `?case=…&answer=…` carries attribution back to the app.
- **5 bias types in MVP**: anchoring, framing, loss aversion, confirmation bias, sunk cost fallacy. Other biases listed in `brief.md` are part of the post-MVP roadmap.
- **LLM cache as cyclic buffer**: English Cases generated by an LLM are stored in a buffer of capacity `N`; new Cases are generated only when the buffer is below threshold or stochastic refresh fires; worst-rated and oldest are evicted.
- **Embedding-based deduplication**: before insertion into the buffer, a candidate Case is embedded and rejected if its cosine similarity to any existing Case exceeds threshold `S`. Prevents 10 paraphrases of the same Case dominating the buffer.
- **LLM-as-judge validation**: every newly generated Case is checked by a second LLM call against the target bias before it can enter the buffer.
- **Seeded gold-standard Cases**: 25 manually authored, validated Cases (5 bias types × 5 variants) seed the database before launch — eliminates cold-start cost.
- **A/B variant Cases** for biases that require contrast (framing, anchoring): the schema supports two variants of the same Case shown to disjoint user groups.
- **Parametric Case structure**: monetary amounts, names, places, time units stored as placeholders (`{currency_amount_small}`, `{local_first_name}`); resolved at translation time from a per-locale dictionary.
- **UI translation — three tiers**:
  - *Tier A (LLM + cache)*: Cases, in-app micro-copy, dynamic stat captions, error messages, tooltips, empty states. On-demand for any language; cached per `(text_hash, target_lang)`.
  - *Tier B (DB-backed AI translation)*: all UI chrome — navigation labels, menu items, button text, headings, form labels, captions, bias-type display names. English source maintained as structured keys (`key`, `title`, `description`) seeded in E020; stored in `ui_string` table with `source_hash`; translated on-demand by AI for any locale; cached in `ui_string_translation` table; automatically re-translated when `source_hash` changes. Frontend fetches `GET /v1/i18n/{locale}` at startup and caches the result.
  - *Tier C (human-curated static)*: landing page marketing copy, legal text (Privacy Policy, ToS, GDPR, cookie banner), Stripe checkout / payment flow text, transactional e-mails (welcome, magic link). MVP languages: `cs` + `en`.
- **Auth**: passwordless magic link only. Anonymous session works without registration for Mode 1.
- **GDPR**: cookie-less analytics, "export my data" + "delete my account" endpoints, audit log of consent.
- **Monetisation**: Stripe Checkout subscription, global price 3.99 € / month (Stripe converts to user's local currency automatically). Free tier = 5 Cases / day; Premium = unlimited + history + per-bias resistance score. B2B billing is **out of MVP scope** (see Non-Goals).
- **Anti-abuse**: per-IP and per-session rate limits, lightweight tracking-cookie-free fingerprint, captcha for free Case generation, idempotency on rating endpoint.
- **Observability**: Sentry for errors, structured JSON logs, Plausible (or PostHog) for funnel analytics, feature flag for Mode 3 launch.
- **Bias-resistance score**: per bias-type rolling percentage of correct answers in the last 20 Cases.
- **Ethics ADR (ADR-002)**: written in E010, defines explicit "we will not sell to" categories (manipulative marketing, dark-pattern designers).
- **Hosting**: production runs on Render.com or Fly.io (decided in ADR-005 in E090). Local development entirely in Docker Compose.

## Non-Goals (explicit out-of-MVP)

The following are valuable, planned post-MVP, and **must not** be included in MVP work:

- **Mode 2** (Comprehensive Test) and any B2B-targeted UI / pricing.
- **B2B sales** of any kind: no Contact-Sales form, no enterprise pricing tier, no multi-tenant infrastructure. B2B starts in Phase E (E140+).
- **Spaced-repetition / Anki-style personalisation** (E160).
- **Streaks, XP, leaderboards** (gamification beyond the bias-resistance score).
- **Friend challenges, social sharing of personal results, "Bias Cards" physical product**.
- **Audio / podcast versions, custom GPT integrations, journaling integrations**.
- **PPP-localised pricing** (price of 2 beers per country) — planned as E165 after the first 100 paying users.
- **Academic validation as gating criterion**: contact a university psychology department during E020/E030, but **launch does not depend on their formal sign-off**. Formal academic validation = E170.
- **More than 5 bias types in MVP**: the remaining biases from `brief.md` are added gradually post-launch as content improvements (not as Epics).
- **Native mobile apps**: web only in MVP. Mobile = post-MVP.
- **Multi-LLM-provider abstraction in MVP**: pick one provider (decided in ADR-003 in E030) and ship; abstraction layer is post-MVP.
- **Reverse-engineering "create-your-own-manipulation" mode** suggested by Gemini.
- **Open data export / public research API** (E180).
- **Other social networks** (LinkedIn, Threads, Bluesky, TikTok). MVP launches on **X / Twitter only**.

## Assumptions

- A single LLM provider (GPT / Claude / Gemini) at API list price is affordable for the projected MVP traffic (≤ 10 k Cases / month after seeding) — to be re-validated in E030 with provider-specific cost projection.
- The 25 seeded gold-standard Cases plus LLM-as-judge filter produce Cases with quality good enough that the median user rating ≥ 3.5 / 5.
- Stripe is available in the operator's country (Czech Republic — confirmed) and supports multi-currency subscriptions for our target markets.
- Render.com or Fly.io free / hobby tier suffices until first 100 paying users; commercial scale concerns are post-launch.
- The operator (Human) handles GDPR / DPO obligations as a sole-trader; no separate DPO infrastructure needed in MVP.
- UI chrome strings (~150 keys, Tier B) have English source authored in E020; AI translations for any locale are generated on-demand starting E040; no manual translation work beyond English source maintenance. Landing page, legal copy, and transactional e-mails (Tier C, ~50 keys) are human-authored in cs + en within E040.
- X / Twitter API access (free or basic tier) is sufficient for ≤ 5 posts / day; if not, we degrade to manual posting via dashboard until budget allows paid tier.
- The user (Human, you) operates as solo developer + AI agent; capacity is variable and time-to-launch is not a hard constraint.
